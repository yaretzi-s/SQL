## 题目1（分组统计+筛选+排序）
### 需求
查询每位用户的**订单总金额**，要求：
1. 输出字段：`user_id`、`user_name`、`total_amount`（总金额）
2. 筛选出总金额 **≥ 300** 的用户
3. 按总金额**降序**排列

```sql
-- 
SELECT u.user_id, u.user_name, SUM(o.order_amount) AS total_amount
FROM users u
JOIN orders o
ON u.user_id = o.user_id
GROUP BY u.user_id, u.user_name
HAVING SUM(o.order_amount) >= 300
ORDER BY total_amount DESC
```

## 题目2（LEFT JOIN 查无数据记录）
### 需求
查询**从未下过任何订单**的用户，输出字段：`user_id`、`user_name`

```sql
-- 
SELECT u.user_id, u.user_name
FROM users u
LEFT JOIN orders o
ON u.user_id = o.user_id
WHERE o.order_id IS NULL
```

## 题目3（重复数据排查）
### 需求
基于 `order_detail` 表，查找**同一订单+同一商品**的重复记录：
1. 输出：`order_id`、`product_id`、`repeat_count`（重复行数）
2. 仅展示重复次数 > 1 的数据
3. 按重复次数**降序**排列

```sql
--
SELECT order_id, product_id, COUNT(*) AS repeat_count
FROM order_detail
GROUP BY order_id, product_id
HAVING COUNT(*) > 1
ORDER BY repeat_count DESC
```

## 题目4（窗口函数排名）
### 需求
统计每个用户总订单金额，使用 `DENSE_RANK` 按总金额做排名（金额相同名次相同）。
输出字段：`ranking`、`user_id`、`user_name`、`total_amount`

```sql
-- 
SELECT DENSE_RANK() OVER(ORDER BY SUM(o.order_amount) DESC) AS ranking, user_id, user_name, SUM(o.order_amount) AS total_amount
FROM users u
JOIN orders o
ON u.user_id = o.user_id
GROUP BY user_id, user_name
ORDER BY ranking```

## 题目5（综合条件查询）
### 需求
统计**男性用户**的订单总金额，只展示总金额大于 150 的男性用户，按总金额升序排列。
输出字段：`user_id`、`user_name`、`total_amount`

```sql
-- 
SELECT u.user_id, u.user_name, SUM(o.order_amount) AS total_amount
FROM users u 
JOIN orders o
ON u.user_id = o.user_id
WHERE u.gender = '男'
GROUP BY u.user_id, u.user_name
HAVING SUM(o.order_amount) > 150
ORDER BY total_amount
```
